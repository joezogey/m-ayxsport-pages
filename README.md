# m-ayxsport-pages

## 项目简介

**m-ayxsport-pages** 是一个用于归档和发布多个独立 HTML 页面的仓库。  
本项目不针对任何特定域名、网站或服务，其目的是以结构化方式存储静态页面资源，便于版本管理、历史回溯和公开访问。

所有页面均为独立的 HTML 文件，不依赖后端服务，可直接在浏览器中打开或通过静态托管服务部署。

## 目录结构

```
m-ayxsport-pages/
├── pages/               # 存放各独立 HTML 页面
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/              # 公用资源文件（CSS、图片、字体等）
│   ├── css/
│   ├── images/
│   └── ...
├── index.html           # 入口导航页（可选）
└── README.md            # 本文件
```

各目录和文件的作用如下：

- **pages/**：核心目录，每个 HTML 页面对应一个独立归档文件。文件名应简洁易懂，便于识别内容。
- **assets/**：存放多个页面可能共用的静态资源，避免重复冗余。
- **index.html**：可选的入口页面，用于汇总展示所有归档页面的链接或说明。

## 页面归档说明

本仓库的页面归档遵循以下原则：

1. **独立性**：每个 HTML 页面自身包含完整的内容与样式，不依赖外部动态数据接口。
2. **版本化**：每次提交对应一个历史版本，支持通过 Git 回滚或对比差异。
3. **可访问性**：页面通过 GitHub Pages 或其他静态托管服务部署后，可直接通过 URL 访问。
4. **无外部依赖**：除非必要，否则页面不引用外部 CDN 资源，确保离线或网络受限环境下仍可正常渲染。

如果您需要添加或更新页面，请直接向 `pages/` 目录提交新的 HTML 文件，并确保文件名不冲突。

## 维护说明

- **提交规范**：建议每次提交仅涉及一个页面的新增或修改，并在提交信息中简要描述变更内容。
- **资源管理**：公用资源（如 CSS 样式、图片等）应放置在 `assets/` 目录下，并在页面中通过相对路径引用。
- **兼容性**：页面应尽量兼容主流浏览器的最新版本，必要时可在页面头部注明兼容性要求。
- **链接检查**：如果页面中包含内部链接（如导航到其他归档页面），请确保链接路径正确，避免死链。
